## 6. 文件命名与版本说明

### 已知版本演化

ESG 主数据存在多个时间点的导出版本：

```
0127 系列  →  数据0127 / qian5000数据0127 / 后5000数据0127
0225 系列  →  新数据0225 / 新数据0225前5000 / 新数据0225后
版本 1 / 版本 2  →  数据版本1_* / 数据版本2_*
重新检查系列  →  美国企业数据重新检查 / 更新_美国企业数据重新检查
```

`前 5000 / 后 5000` 是按公司分批拉取（Eikon 单次请求上限）。如使用，建议：

- 公司清单 / 基础属性：以 `更新_美国企业数据重新检查.xlsx` 为准
- ESG 评分（旧版本）：合并 `qian5000数据0127.xlsx` + `后5000数据0127.xlsx`
- ESG 评分 + 环境支出（新版本）：合并 `新数据0225前5000.xlsx` + `新数据0225后.xlsx`

### 待处理标记

`EU Partners/` 中文件名后缀含义：

- `(not done)`：尚未完成数据拉取
- `(error)`：拉取过程中报错，需要重试

集中在美国（58 个文件中过半带后缀）和日本（26 个文件中含多个 error）两国。

### Sheet 命名约定

- 中文 sheet（`美国` / `欧盟` / `中国` / `意大利`）：人工命名的最终结果表
- base64 hash sheet（如 `ODk2MTRkMWQtZjU1NS00ZG`）：Eikon 拉取过程的中间临时表，**实际数据建议从中文 sheet 读取**
- `Search Results` + `Sheet1`：EU Partners 标准两段式结构（清单 + 时间序列）
